# Constraints

Store versioned Neo4j constraint assets here when they are externalized from
runtime/demo code.

## Current concrete asset

- `current_demo_constraints_posture.md`: documents the current accepted posture
	that there is no stable checked-in demo-owned constraint bundle yet, and that
	the local candidate graph can operate without one.

At the current checkpoint, this folder is primarily documenting constraint
ownership and promotion rules rather than storing executable constraint files.
